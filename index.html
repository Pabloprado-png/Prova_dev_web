A importância de ter uma boa prática de programação em desenvolvimento web

Um dos maiores erros ao iniciar na programação web é subestimar o aprendizado de conceitos básicos, ou seja, conceitos sobre html, css e javascript.
Antes de começarmos na prática como desenvolvedores, primeiro pensamos em como seria a experiência como usuário da nossa própria aplicação.
Devemos sempre nos perguntar se o método que iremos implementar é realmente didático para que possamos deixar cada vez mais explicito o que queremos propor dentro do site.

❝ Esse texto realmente fica bom aqui? ❞
❝ Será que essa cor combina com aquela? ❞
❝ Vou ter dificuldade para acessar esse conteúdo? ❞

Essas perguntas ampliam completamente nosso pensamento crítico sobre a usabilidade e eficiência das aplicações e ajudam a melhorar nossa capacidade em ter uma boa base teórica sobre esses conceitos.
Na prática, nossa maior dificuldade é nos adaptarmos à diversas plataformas e dispositivos com diferentes telas.
Um conceito que chegou para competir com o design responsivo e têm revolucionado o mercado, é o Mobile First, onde o foco inicial da arquitetura e desenvolvimento é direcionado aos dispositivos móveis e em seguida para os desktops.

Diferenças entre Design responsivo e Mobile First

A diferença entre responsivo e mobile é que a versão mobile é uma ótima opção para portais e sites que têm sistemas ou funcionalidades específicas que não podem funcionar da mesma forma que em desktops.
Já o responsivo, é bom para sites que não tem alguma funcionalidade que pode ser utilizada em telas grandes.
Além do conteúdo ser mais fácil de ser trabalhado para diferentes dispositivos, nenhuma informação é perdida quando alterado para determinados formatos, mas ambos são boas práticas de desenvolvimento, tendo isso em mente, devemos nos fundamentar antes em toda teoria para podermos otimizar o site e garantir que o usuário tenha uma ótima experiência.
